# Release `0.4.0-alpha.5`

Date: `2026-04-23`

## Summary

This release pushes Epsilon past a purely backend-oriented emulator slice and into a full product surface:

- public CMS
- launcher backend
- native launcher app
- provisional published client

The key architectural rule is now explicit across the stack:

- CMS does not equal game
- launcher does not equal hotel presence
- only emulator-confirmed runtime presence counts as real entry

## Included

### CMS

- homepage
- login
- register
- authenticated launcher access choice
- launcher access-code generation
- hotel/news/photos/community/support payloads

### Launcher backend

- desktop config endpoint
- update channels
- launch profile selection
- launcher code redeem
- client-started telemetry

### Desktop app

- Avalonia/VB launcher integrated as `apps/epsilon-launcher-native`
- macOS `.app` output
- macOS `.dmg` output

### Runtime

- room tick scheduler
- room animation slice
- first roller support
- collector platform slice

## Known instability

- CMS still needs another hardening pass
- launcher app is operational but not yet polished/stable
- `web-alpha` is provisional
- final Unity/Nitro desktop packages are not published
- durability is still limited by `InMemory` dependencies

## Release decision

This is a valid alpha release because the architecture and product flow are now coherent, but it should not be treated as a stable public launch.
